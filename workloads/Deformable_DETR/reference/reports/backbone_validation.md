# Backbone Validation Report

**Generated:** 2026-02-18 11:13:07

**Test Suite:** PyTorch vs TTSim Backbone Implementation Comparison

This report compares the PyTorch and TTSim implementations of the Deformable DETR backbone modules.

## Test Approach
- **FrozenBatchNorm2d**: Full numerical validation (8x8 inputs)
- **ResNetBottleneck**: Full numerical validation with pretrained weights (8x8 inputs)
- **Backbone**: Shape + mask validation only (32x32 inputs). Numerical redundant — block-level proven in Test 2
- **Joiner**: Shape + mask for feature maps. Full numerical for positional encoding (sine, weight-free numpy)

**Rationale:** TTSim Conv2d runs through nested-loop numpy (data_compute.py), making full-backbone
numerical validation both prohibitively slow and redundant since the building blocks are individually validated.
